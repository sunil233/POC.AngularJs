The Promised Land (in Angular)

 
 

A Promise in Short:is the future result of an operation or a placeholder for a successful result or an error

 
 
  • Powerfull for chaining promises via .then()
  • The ability to handle multiple asynchronous calls sequentially
  • Especially useful when one operation depends on information from anothet operation (eg: a rest call depending on information from another rest call)
 

$q

 
 
  • Light Weight implementation of Q
  • $http,$routeProvider and $timeout all uses Promises
 
 

$q.all()

 
 
  • Handling multiple Promises (Parallel)
  • $q.all resolves when all promises have been successfully resolved
  • $q.all rejects as soon as one promise has been rejected,returning error message
